# 🍽️ 聚餐自動分組(桌)系統 | BanquetTable Grouping System

這是一個專為聚餐、尾牙、婚宴設計的**自動隨機分組與分桌系統**。本系統採用純前端技術開發，外觀精美，並完美支援直接部署於 **GitHub Pages**。

為了滿足不同場景，本系統提供兩種資料儲存與同步模式：
1. **單機模式 (LocalStorage)**：適合現場放置一台平板或手機在簽到桌，由參加人員輪流輸入姓名登記。
2. **雲端即時同步模式 (Supabase)**：管理者在設定頁面輸入免費的 Supabase API 資訊，系統會產生**專屬的參加者連結**。參加者可用各自的手機掃碼或點擊連結輸入姓名，系統將即時隨機分組並同步至所有人的畫面！

---

## ✨ 核心特色

- 🎨 **極致奢華視覺**：奢華暗紫與琥珀金配色，完美契合宴會氣氛，搭配玻璃擬態與平滑動畫。
- 🎲 **智能隨機分組**：自動計算尚有空位的桌次進行隨機分配，公平有趣。
- 🔒 **防重防改機制**：
  - 姓名唯一性校驗，防止重複登記。
  - 綁定裝置瀏覽器快取，登記成功後無法更改姓名或重新登記，確保結果公正。
- ⚡ **即時同步**：在雲端模式下，參加者一鍵登入，管理者大螢幕立即顯示分組名單。
- 📥 **CSV 資料下載**：管理者可一鍵下載所有參加者的分組名單，方便後續列印與點名。

## ☁️ 雲端同步設定指南 (Supabase)

如果您想讓參加者**使用各自的手機進行登記**，請按照以下步驟免費設定 Supabase 資料庫（只需 2 分鐘）：

### 第一步：註冊與建立專案
1. 前往 [Supabase 官網](https://supabase.com/) 並註冊/登入帳號。
2. 點擊 **New Project**，輸入專案名稱（例如 `banquet-table`）與設定資料庫密碼，選擇離您最近的伺服器區域，然後點擊 **Create new project**。

### 第二步：建立資料表 (SQL)
1. 專案建立完成後，在左側選單點擊 **SQL Editor** ⚙️，再點擊 **New query**。
2. 將以下 SQL 語法貼入編輯器中，並點擊 **Run** 執行：

```sql
-- 建立參加者資料表
create table public.participants (
  id bigint generated by default as identity primary key,
  event_id text not null,
  name text not null,
  table_number integer not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  
  -- 設定同一個活動內姓名不可重複
  constraint unique_event_name unique (event_id, name)
);

-- 啟用 Row Level Security (RLS) 或為了簡便直接開放匿名寫入 (建議在聚餐時開啟)
alter table public.participants enable row level security;

-- 建立所有人皆可讀寫的安全性原則 (無限制讀寫，方便宴會即時登記)
create policy "Allow public read" on public.participants for select using (true);
create policy "Allow public insert" on public.participants for insert with check (true);
```

### 第三步：取得 API 金鑰並開始使用
1. 在左側選單點擊 **Project Settings** ⚙️，選擇 **API**。
2. 找到並複製以下兩個資訊：
   - **Project URL** (例如 `https://xxxxxx.supabase.co`)
   - **anon public** (例如 `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`)
3. 打開本應用程式的 **設定頁面**，切換為「雲端同步模式」，將這兩項資訊填入。
4. 設定桌數與每桌人數後，點擊 **「產生並複製分享連結」** 發送給您的好友與賓客！

## 🚀 部署到 GitHub Pages

1. 將本專案的所有檔案（`index.html`、`style.css`、`app.js`、`README.md`）上傳至您的 GitHub 儲存庫（Repository）。
2. 進入儲存庫的 **Settings** -> **Pages**。
3. 在 **Build and deployment** 下的 Source 選擇 **Deploy from a branch**，Branch 選擇 `main` (或 `master`) 與 `/ (root)`，然後點擊 **Save**。
4. 約 1 分鐘後，即可透過 `https://<您的帳號>.github.io/<儲存庫名稱>/` 存取您的分組系統！
